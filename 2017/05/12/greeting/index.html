跌跌撞撞来建站

Posté le 2017-05-12 | In Thoughts

建了一个站，踩了两个坑

此文命途多舛

为了建立这个博客，前后花了至少4个小时。Github Pages顺利，Hexo 建立本地博客站点顺利，但是为了布署的时侯，出了问题。我试图将这个本地博客 git push 到 git@github.com:username/username.github.io。照着这个思路，走了很多弯路啊。

坑：消失的git

按照Github Pages的指引 建立一个实验站点，成功建立一个空架子。此时只有一个源文件index.html

接下来将这个repo 克隆到本地，然后在里面执行

hexo init

然后git就消失了。

接下来

hexo g
hexo s

都是正常的的。localhost:4000也成功显示本地站点。

到这样，如果配置_config.yml，执行

hexo d

就成功，一切都结束了。
但是按照我的错误思路就……

坑：消失的站

接下来

git remote add origin git@github.com:username/username.github.io
git push origin master

这里还遇到了冲突，以为远程只有一个index.html, 本地确实一个博客。强行覆盖：

git push origin master -f

最后我成功地将本地的博客站push 到了username.github.io, 然后再访问username.github.io 就只能看到错误提示。

祭奠我死去的四个小时

RTFM

出现这种问题，最主要的原因是胡乱搜索，按照网上的教程, 有的教程没有提到布署这一步，不幸的是，我参看的几篇正好都是。

事实上，hexo 的文档并不复杂，通读一遍，完全没有问题。

而且就在本地博客建立成功的，显示的 “Hello World” 的页面提到四个常用命令里有

hexo deploy

还能说什么呢：

灯下黑

对静态网站的知识都快忘光了

按照这条错误思路走的时侯，总是感觉哪里不对，最后也是发觉这个思路有问题，才重返正轨的。
